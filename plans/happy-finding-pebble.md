# AI 换衣应用 - 产品计划

## 产品概述

一款时尚简约风格的 AI 虚拟试穿应用。用户上传个人照片，从服装库中选择心仪服装，AI 生成换衣效果图。

---

## 设计风格

- **灵感来源**：ZARA、COS、Everlane 等时尚品牌官网
- **配色方案**：纯净白底 + 温暖珊瑚色（Coral）作为主色调，搭配金色点缀
- **字体**：Playfair Display（标题，时尚编辑感）+ Inter（正文，清晰现代）
- **特点**：大量留白、大图片展示、圆润按钮、微妙动画过渡

---

## 核心功能 (v1)

### 1. 品牌首页
- 全屏视觉 Hero 区域，时尚大图背景
- 品牌标语 + "开始试穿" 行动按钮
- "使用方法" 三步引导（上传 → 选择 → 生成）
- 产品特色展示区

### 2. 照片上传
- 拖拽/点击上传人物照片
- 上传后即时预览，支持更换
- 拍照小贴士卡片（全身照、光线好、背景简洁）
- 文件类型与大小校验，错误提示

### 3. 服装选择
- 分类筛选栏（上装、下装、连衣裙、外套、配饰）
- 响应式服装卡片网格，悬停动画
- 选中状态高亮标识
- 快速预览弹窗查看大图和详情
- 左侧小窗显示已上传的人物照片

### 4. 试穿生成
- 左右分栏展示：人物照片 vs 选中服装
- 醒目的 "生成试穿效果" 按钮
- 优雅的加载动画 + 进度条
- 分步文字提示（"分析照片中..." → "应用服装..." → "优化细节..."）

### 5. 结果展示
- 前后对比视图（原图 vs 效果图）
- 下载效果图
- "换件衣服试试" 快捷操作
- "重新开始" 返回首页

## 用户流程

```
首页 → 上传照片 → 选择服装 → 生成效果 → 查看结果
  ↑                              ↓
  ←←←←←←← 重新开始 ←←←←←←←←←←←←
```

每个步骤顶部显示进度指示器（步骤 1/3、2/3、3/3）。

## 设计系统

### 色彩 (HSL)
- **主色 (Coral)**：`8 65% 65%` — 温暖珊瑚色，所有 CTA 按钮
- **背景**：`0 0% 100%` — 纯白
- **文字**：`0 0% 8%` — 近黑色
- **辅助色**：`30 15% 96%` — 暖灰
- **点缀金**：`38 60% 62%` — 优雅金色

### 字体
- 标题：Playfair Display（衬线体，时尚感）
- 正文：Inter（无衬线，现代清晰）

### 动画
- 页面切换：淡入 + 上滑
- 卡片悬停：微缩放 + 阴影
- 网格加载：依次出现（stagger）
- 加载进度：闪光（shimmer）效果

### 按钮样式
- 主要 CTA：珊瑚色圆角胶囊按钮
- 次要操作：描边按钮
- 辅助操作：Ghost 按钮

## 模拟数据

- 服装库约 20 件商品，覆盖 5 个分类
- 使用 Unsplash 高质量时尚图片
- AI 生成模拟 3~5 秒延迟 + 进度回调
- 后续用户提供真实 API 后，只需替换 `mockApi.ts` 即可

## 实施顺序

1. **基础设施**：设计系统（index.css + tailwind.config.ts）、类型定义、Zustand Store
2. **数据层**：模拟服装数据、模拟 API、应用常量
3. **布局框架**：Header、Footer、路由配置
4. **首页**：Hero、使用步骤、功能特色
5. **上传页**：拖拽上传、照片预览、拍照提示
6. **服装库页**：分类筛选、服装网格、快速预览
7. **试穿页**：工作区、加载动画、结果展示
8. **收尾**：SEO meta 标签、NotFound 页面适配

---

## 验证方式

1. 启动开发服务器 `pnpm run dev` 确认无构建错误
2. 访问首页，检查 Hero 展示和导航
3. 完整走一遍流程：上传照片 → 选择服装 → 生成 → 查看结果
4. 验证响应式布局（移动端 + 桌面端）
5. 检查所有 Toast 提示和错误处理
6. 确认 "重新开始" 和 "换件衣服" 流程正常
